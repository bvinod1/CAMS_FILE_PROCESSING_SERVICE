# Phase 1: Data Model - US-101: Secure File Upload Initiation

**Purpose**: Define the data entities required for this feature.
**Created**: 2026-03-08
**Plan**: [plan.md](plan.md)

This document outlines the structure of the data entities based on the feature specification.

## Entity: FileRecord

Represents the state and metadata of a single file throughout its processing lifecycle. This entity is persisted in Google Cloud Spanner.

**Table Name**: `file_records`

| Attribute | Data Type (Spanner) | Description | Constraints / Rules |
|---|---|---|---|
| `fileId` | `STRING(36)` | **Primary Key**. A unique identifier (UUID) for the file record, generated by the service. | NOT NULL |
| `originalFileName` | `STRING(1024)` | The original name of the file provided by the user. | NOT NULL |
| `flowType` | `STRING(255)` | The business process category (e.g., "NAV", "Transactions"). Used to select validation templates. | NOT NULL |
| `checksum` | `STRING(255)` | The client-provided checksum (e.g., MD5 or SHA256) for integrity validation post-upload. | NOT NULL |
| `status` | `STRING(50)` | The current state of the file in the processing workflow. Initial status is `AWAITING_UPLOAD`. | NOT NULL |
| `createdAt` | `TIMESTAMP` | Timestamp when the record was created (when the upload URL was requested). | NOT NULL |
| `updatedAt` | `TIMESTAMP` | Timestamp of the last modification to the record. | NOT NULL |
| `gcsBucket` | `STRING(1024)` | The GCS bucket where the file is stored. | |
| `gcsObjectPath` | `STRING(2048)` | The full path to the file object within the GCS bucket. | |

### Spanner DDL

```sql
CREATE TABLE file_records (
    fileId STRING(36) NOT NULL,
    originalFileName STRING(1024) NOT NULL,
    flowType STRING(255) NOT NULL,
    checksum STRING(255) NOT NULL,
    status STRING(50) NOT NULL,
    createdAt TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp=true),
    updatedAt TIMESTAMP NOT NULL OPTIONS (allow_commit_timestamp=true),
    gcsBucket STRING(1024),
    gcsObjectPath STRING(2048),
) PRIMARY KEY (fileId);

CREATE INDEX file_records_by_status ON file_records(status);
```
