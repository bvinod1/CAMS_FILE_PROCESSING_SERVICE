# Phase 1: Data Model - US-101: Secure File Upload Initiation

**Purpose**: Define the data entities required for this feature.
**Created**: 2026-03-08
**Updated**: 2026-03-15 (amended for Epic 2 alignment)
**Plan**: [plan.md](plan.md)

This document outlines the structure of the data entities based on the feature specification.

## Entity: FileRecord

Represents the state and metadata of a single file throughout its processing lifecycle. This entity is persisted in Google Cloud Spanner.

**Table Name**: `file_records`

| Attribute | Data Type (Spanner) | Description | Constraints / Rules |
|---|---|---|---|
| `fileId` | `STRING(36)` | **Primary Key**. UUID generated by the service. | NOT NULL |
| `originalFileName` | `STRING(1024)` | The original name of the file provided by the user. | NOT NULL |
| `flowType` | `STRING(255)` | Business process category (e.g. `NAV`, `TRANSACTIONS`). | NOT NULL |
| `checksum` | `STRING(255)` | Client-supplied MD5 (32 hex) or SHA-256 (64 hex) checksum. | NOT NULL |
| `checksumSha256` | `STRING(64)` | System-computed SHA-256 after file lands in quarantine. Set by scan worker. | |
| `ingressChannel` | `STRING(32)` | How the file entered: `REST`, `SFTP`, `GCS_TRIGGER`. | NOT NULL, default `REST` |
| `priority` | `INT64` | Processing priority. 0=highest (NAV), 1=standard, 2=bulk. | NOT NULL, default 1 |
| `status` | `STRING(50)` | Current lifecycle state. Must be a valid `FileStatus` enum value. | NOT NULL, see status table below |
| `createdAt` | `TIMESTAMP` | When the upload URL was requested. | NOT NULL |
| `updatedAt` | `TIMESTAMP` | Last state change timestamp. | NOT NULL |
| `scannedAt` | `TIMESTAMP` | When the ClamAV scan completed. Null until scanning finishes. | |
| `scanId` | `STRING(36)` | FK to `scan_results.scan_id`. Null until scan completes. | |
| `gcsBucket` | `STRING(1024)` | The storage bucket where the file resides. | |
| `gcsObjectPath` | `STRING(2048)` | Full object path within the bucket. | |
| `errorMessage` | `STRING(MAX)` | Last terminal error description. | |

### Valid `status` Values

| Status | Set By | Meaning |
|---|---|---|
| `AWAITING_UPLOAD` | Upload Service | Pre-signed URL issued; waiting for client upload |
| `UPLOADED` | Upload Service (confirm endpoint) | Client confirmed upload complete; FileReceivedEvent published |
| `SCANNING` | Scan Worker (Epic 2) | ClamAV scan in progress |
| `SCANNED_CLEAN` | Scan Worker (Epic 2) | File is clean; ready for validation |
| `QUARANTINED` | Scan Worker (Epic 2) | Virus detected; file never leaves quarantine |
| `SCAN_ERROR` | Scan Worker (Epic 2) | ClamAV unreachable or unexpected response |
| `VALIDATING` | Validation Worker (Epic 3) | Header/structure validation in progress |
| `VALIDATED` | Validation Worker (Epic 3) | Validation passed |
| `VALIDATION_FAILED` | Validation Worker (Epic 3) | Header/structure invalid |
| `PROCESSING` | Processing Worker (Epic 4) | Spring Batch record processing in progress |
| `COMPLETED` | Processing Worker (Epic 4) | All records processed |
| `PARTIALLY_COMPLETED` | Processing Worker (Epic 4) | Some records succeeded, some failed |
| `FAILED` | Any worker | Unrecoverable error; terminal state |

### Spanner DDL (updated)

```sql
CREATE TABLE file_records (
    fileId           STRING(36)    NOT NULL,
    originalFileName STRING(1024)  NOT NULL,
    flowType         STRING(255)   NOT NULL,
    checksum         STRING(255)   NOT NULL,
    checksumSha256   STRING(64),
    ingressChannel   STRING(32)    NOT NULL DEFAULT ('REST'),
    priority         INT64         NOT NULL DEFAULT (1),
    status           STRING(50)    NOT NULL
        CHECK (status IN (
            'AWAITING_UPLOAD','UPLOADED','SCANNING','SCANNED_CLEAN',
            'QUARANTINED','SCAN_ERROR','VALIDATING','VALIDATED',
            'VALIDATION_FAILED','PROCESSING','COMPLETED',
            'PARTIALLY_COMPLETED','FAILED'
        )),
    createdAt        TIMESTAMP     NOT NULL OPTIONS (allow_commit_timestamp=true),
    updatedAt        TIMESTAMP     NOT NULL OPTIONS (allow_commit_timestamp=true),
    scannedAt        TIMESTAMP,
    scanId           STRING(36),
    gcsBucket        STRING(1024),
    gcsObjectPath    STRING(2048),
    errorMessage     STRING(MAX),
) PRIMARY KEY (fileId);

CREATE INDEX file_records_by_status       ON file_records(status);
CREATE INDEX file_records_by_flow_type    ON file_records(flowType);
```

### IAM Bucket Access Rules

Enforced at infrastructure level — **these are non-negotiable security rules**:

| Service | Quarantine Bucket | Processing Bucket |
|---|---|---|
| Upload Service | WRITE only (pre-signed URL scoped) | ❌ No access |
| Scan Worker | READ only | WRITE only (copy clean files) |
| Validation Worker | ❌ No access | READ only |
| Processing Worker | ❌ No access | READ only |
| Any other service | ❌ No access | ❌ No access |
